Bootstrap 5 Input Groups - Complete Notes

1. Basic Input Groups

The .input-group class is a container that enhances an input field by adding an icon, text, or button in front of or behind the input field as "help text". Input groups are useful for adding context or additional information to form inputs.

Structure and Classes:

  • .input-group - Container wrapper for the entire input group
  • .input-group-text - Class for text/icon elements added to the input (addons)
  • .form-control - The actual input field
  • Addons can be placed before (prepend) or after (append) the input

Key Points:

  • Always wrap input groups in a .input-group container
  • Use .input-group-text for any text or symbols added to the group
  • The .form-control class is required for the input field
  • Text can be prepended (before) or appended (after) the input
  • Multiple addons can be used on the same input group

2. Input Group Size

Control the size of input groups using size modifier classes. Size classes apply to the entire input group including the text addons and input field.

Size Classes:

  • .input-group-sm - Small input group
  • .input-group - Default normal-sized input group
  • .input-group-lg - Large input group

JavaScript Code for this Example:
// Get references to the input and button
const input = document.getElementById('selectFromListInput');
const btn = document.getElementById('selectFromListBtn');
const menuItems = document.querySelectorAll('#selectFromListMenu .dropdown-item');

// Add click listener to each dropdown item
menuItems.forEach(item => {
  item.addEventListener('click', function(e) {
    e.preventDefault(); // Prevent default link behavior
    
    // Get the selected value from data attribute
    const selectedValue = this.getAttribute('data-value');
    
    // Update input field with selected value
    input.value = selectedValue;
    
    // Update button text to show selection
    btn.textContent = selectedValue;
  });
});
How It Works:
  • Each dropdown item has a data-value attribute with the value to be inserted
  • JavaScript listens for clicks on each dropdown item
  • When clicked, the input field is populated with the selected value
  • The button text is updated to show the last selected option
  • e.preventDefault() prevents the default link navigation

Best Practices for Input Groups

  • Always wrap input groups in a .input-group container
  • Use .input-group-text for all text or icon addons
  • Apply the .form-control class to all input fields
  • Keep addons short and clear for better usability
  • Use consistent size modifiers throughout your form
  • Ensure buttons and addons are visually distinct from input fields
  • Use aria-label for accessibility when using checkboxes or radios in input groups
  • Consider using descriptive placeholders in input fields
  • Test on different screen sizes to ensure input groups remain usable
  • Don't overcrowd input groups - keep the number of elements reasonable
  • Use semantic button types (type="button", type="submit")
  • Ensure proper color contrast for text addons and input fields
