Bootstrap 5 Select - Complete Notes

1. Select Menu

Select menus are used when you want to allow the user to pick from multiple options. To style a select menu in Bootstrap 5, add the .form-select class to the <select> element.

Multiple Select Example:

2. Select Menu Size

Control the size of select menus using size modifier classes:

  • .form-select-lg - Creates a large select menu
  • .form-select - Default normal-sized select menu
  • .form-select-sm - Creates a small select menu

4. Data Lists

Bootstrap also styles data lists, which provide a list of pre-defined options for an <input> element. Use the <datalist> element with <option> children. Connect the input to the datalist using the list attribute that matches the datalist's id.

How it works:

  • Create a <datalist> element with a unique id
  • Add <option> elements inside the datalist with value attributes
  • Add the list attribute to an <input> element pointing to the datalist's id
  • Apply .form-control class to the input for proper styling
  • Apply .form-label class to the associated label

Key Bootstrap Select Classes Summary

Class Purpose
.form-select Styles select elements with proper Bootstrap appearance
.form-select-lg Creates a large select menu
.form-select-sm Creates a small select menu
.form-control Styles input elements used with datalists
.form-label Styles label elements with proper padding

HTML Attributes for Select Elements

Attribute Purpose Example
multiple Allows selection of multiple options <select multiple>
disabled Disables the select menu <select disabled>
id Unique identifier for the select element <select id="mySelect">
name Form field name for submission <select name="category">
list Connects input to a datalist (for inputs) <input list="browsers">

Best Practices for Bootstrap Select Elements

  • Always use .form-select class on <select> elements for consistent styling
  • Use .form-label class on associated labels
  • Wrap form groups in containers with spacing classes like .mb-3
  • Use meaningful option text that clearly describes what the option represents
  • Consider using datalists for searchable lists instead of long select menus
  • Make sure labels and form controls have proper associations using for and id attributes
  • Test accessibility with keyboard navigation
  • Use size modifiers consistently throughout your form
  • Provide default or placeholder text to guide users
