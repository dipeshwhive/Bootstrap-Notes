Bootstrap 5 Checkboxes and Radio Buttons - Complete Notes

1. Checkboxes

Checkboxes are used when you want the user to select any number of options from a list of preset options. Users can select zero, one, or multiple checkboxes.

Structure and Classes:

  • .form-check - Wrapper container that ensures proper margins for labels and checkboxes
  • .form-check-input - Class applied to the checkbox input element for proper styling
  • .form-check-label - Class applied to the label element
  • checked - HTML attribute to make a checkbox checked by default

Key Points:

  • Always wrap checkboxes in a .form-check container
  • Use id on the input and for on the label to link them together
  • Use the checked attribute to pre-select a checkbox
  • Use the disabled attribute to disable a checkbox
  • The name attribute is used for form submission

2. Radio Buttons

Radio buttons are used when you want to limit the user to just one selection from a list of preset options. Only one radio button in a group can be selected at a time.

Structure and Classes:

  • .form-check - Wrapper container for each radio button
  • .form-check-input - Class applied to the radio button input element
  • .form-check-label - Class applied to the label element
  • name - All radio buttons in a group must have the same name attribute
  • checked - HTML attribute to make a radio button selected by default

Key Points:

  • All radio buttons in a group must have the same name attribute
  • Each radio button must have a unique id and a corresponding label with for
  • Only one radio button in a named group can be selected at a time
  • The value attribute specifies what is submitted when that radio button is selected
  • Use checked to pre-select a radio button (should only have one per group)
  • Use disabled to disable a radio button

3. Toggle Switches

Toggle switches are a stylistic variation of checkboxes that look like physical on/off switches. They use a checkbox with the .form-switch class added to the .form-check container.

Structure and Classes:

  • .form-check - Base wrapper container
  • .form-switch - Additional class to style checkbox as a toggle switch
  • .form-check-input - Class applied to the checkbox input
  • .form-check-label - Class applied to the label element

Key Points:

  • Add .form-switch class to the .form-check container to convert a checkbox to a toggle switch
  • Toggle switches are still checkboxes underneath, so they work the same way in forms
  • Multiple toggle switches are independent and can all be on, all be off, or in any combination
  • Use checked to make a toggle switch appear enabled by default
  • Use disabled to disable a toggle switch
  • The visual design looks like a physical switch which is more intuitive for on/off states

Checkboxes vs Radio Buttons vs Toggle Switches

Feature Checkboxes Radio Buttons Toggle Switches
Selection Type Multiple selections allowed Single selection only Binary on/off state
Base HTML Element <input type="checkbox"> <input type="radio"> <input type="checkbox">
Bootstrap Classes .form-check, .form-check-input .form-check, .form-check-input .form-check, .form-switch, .form-check-input
Grouping No grouping required Must have same name attribute No grouping required
Use Case Multiple options, all may apply Choose one from list Enable/disable a single feature
Visual Style Square checkboxes Round radio buttons On/off switch appearance

Key Bootstrap Classes Summary

Class Purpose
.form-check Base wrapper container for checkboxes and radio buttons
.form-check-input Styles the checkbox or radio button input element
.form-check-label Styles the label element with proper spacing
.form-switch Converts a checkbox to a toggle switch (used with .form-check)

HTML Attributes for Checkboxes, Radio Buttons, and Switches

Attribute Purpose Example
type Specifies the input type type="checkbox" or type="radio"
name Form field name (all radio buttons in a group must share this) name="preference"
id Unique identifier for the input element id="option1"
value Value submitted when selected value="yes"
checked Makes the input selected/enabled by default checked
disabled Disables the input element disabled
for (on label) Associates label with input via input's id <label for="option1">

Best Practices for Checkboxes, Radio Buttons, and Switches

  • Always wrap form controls in a .form-check container
  • Always use the .form-check-input and .form-check-label classes
  • Use unique id attributes on all inputs and corresponding for attributes on labels
  • Group related radio buttons by using the same name attribute
  • For multiple selections, use checkboxes; for single selection, use radio buttons
  • Use toggle switches for binary on/off states (e.g., enable/disable a feature)
  • Make sure the clickable area includes both the input and label for better usability
  • Use the checked attribute to show the default state
  • Consider accessibility - ensure proper label associations for screen readers
  • Only disable options when necessary, as it can confuse users
  • Provide clear, descriptive label text that explains what each option does
  • Test on different browsers and devices to ensure consistent appearance
